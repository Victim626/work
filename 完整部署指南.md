# 游泳教练辅助平台 — 完整部署指南

## 技术栈总览

| 层级 | 技术 |
|------|------|
| 后端 | Spring Boot 3.5 + MyBatis + JDK 17 |
| 前端 | Vue 3 + Vite 5 + TypeScript |
| 数据库 | MySQL 8.x |
| 反向代理 | Nginx |

---

## 一、服务器环境准备

### 1.1 安装 JDK 17

**Linux (Ubuntu/Debian):**

```bash
sudo apt update
sudo apt install openjdk-17-jdk -y
java -version
```

**Linux (CentOS/RHEL):**

```bash
sudo yum install java-17-openjdk java-17-openjdk-devel -y
java -version
```

### 1.2 安装 Maven 3.8+

```bash
# Ubuntu
sudo apt install maven -y

# CentOS
sudo yum install maven -y
```

或使用项目自带的 `./mvnw`，无需单独安装。

### 1.3 安装 Node.js 18+

```bash
# 使用 NodeSource
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install nodejs -y

node -v
npm -v
```

### 1.4 安装 MySQL 8.x

```bash
# Ubuntu
sudo apt install mysql-server -y
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全初始化，设置 root 密码
sudo mysql_secure_installation
```

### 1.5 安装 Nginx

```bash
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
```

## 二、数据库部署

### 2.1 创建数据库并导入数据

将项目中的 `swimming_coach_platform_schema.sql` 上传到服务器，然后执行：

```bash
mysql -u root -p < swimming_coach_platform_schema.sql
```

该脚本会自动创建 `swimming_coach_platform` 数据库、18 张表和完整测试数据。

如需手动创建：

```sql
CREATE DATABASE swimming_coach_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 验证

```bash
mysql -u root -p -e "USE swimming_coach_platform; SHOW TABLES;"
```

## 三、后端部署

### 3.1 上传项目代码

```bash
# 方式一：git clone
git clone https://github.com/Victim626/swimming.git
cd swimming

# 方式二：直接上传项目目录到服务器
scp -r ./swimming user@server_ip:/opt/swimming
```

### 3.2 修改数据库配置

编辑 `src/main/resources/application.properties`：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/swimming_coach_platform?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=你的MySQL密码
```

其他配置保持默认即可：

- 端口：`8081`
- 上下文路径：`/mySpringBoot`

### 3.3 打包

```bash
cd /opt/swimming
./mvnw clean package -DskipTests
```

打包产物在 `target/swimming-0.0.1-SNAPSHOT.jar`。

### 3.4 启动

```bash
# 前台启动（调试用）
java -jar target/swimming-0.0.1-SNAPSHOT.jar

# 后台启动（生产用）
nohup java -jar target/swimming-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

### 3.5 验证

```bash
curl http://localhost:8081/mySpringBoot
```

### 3.6 设置 systemd 服务（推荐）

创建 `/etc/systemd/system/swimming.service`：

```ini
[Unit]
Description=Swimming Coach Platform Backend
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/swimming
ExecStart=/usr/bin/java -jar /opt/swimming/target/swimming-0.0.1-SNAPSHOT.jar
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable swimming
sudo systemctl start swimming
sudo systemctl status swimming
```

日志查看：

```bash
journalctl -u swimming -f
```

## 四、前端部署

### 4.1 安装依赖并打包

```bash
cd /opt/swimming/frontend
npm install
npm run build
```

打包产物在 `frontend/dist/` 目录。

### 4.2 修改前端 API 地址

生产环境中，前端通过 Nginx 反向代理访问后端，API 地址保持相对路径 `/mySpringBoot` 即可，不需要修改。只有前后端分开部署在不同域名时才需要修改。

## 五、Nginx 配置

### 5.1 创建配置文件

```bash
sudo vi /etc/nginx/sites-available/swimming
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;   # 例如 192.168.1.100 或 yourdomain.com

    # 前端静态文件
    location / {
        root /opt/swimming/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /mySpringBoot/ {
        proxy_pass http://127.0.0.1:8081/mySpringBoot/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持（项目用到了 WebSocket）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 5.2 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/swimming /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default   # 移除默认站点（可选）

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 九、常见问题

| 问题 | 排查方法 |
|------|----------|
| 数据库连不上 | 检查 MySQL 服务状态、密码、数据库名是否正确 |
| 后端启动失败 | 检查 Java 版本（需 17）、端口 8081 是否被占用、数据库表是否已创建 |
| 前端页面空白 | 检查 Nginx root 路径是否指向 `dist`、`try_files` 是否配置 |
| API 404 | 检查 Nginx `location /mySpringBoot/` 和后端 `context-path` 是否一致 |
| WebSocket 不通 | 确认 Nginx 配置了 `Upgrade` 和 `Connection` 头 |

## 十、更新部署流程

代码更新后重新部署：

```bash
cd /opt/swimming
git pull

# 重新打包后端
./mvnw clean package -DskipTests
sudo systemctl restart swimming

# 重新打包前端
cd frontend
npm install
npm run build
# Nginx 无需重启，静态文件已更新
```

## 附录：关键配置文件一览

### application.properties

```properties
spring.application.name=swimming

spring.datasource.url=jdbc:mysql://localhost:3306/swimming_coach_platform?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=

mybatis.mapper-locations=classpath:mapper/*.xml
mybatis.type-aliases-package=com.usbtj.springboot.swimming.entity
mybatis.configuration.map-underscore-to-camel-case=true
```

### vite.config.ts

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 80,
    host: '0.0.0.0',
    proxy: {
      '/mySpringBoot': {
        target: 'http://127.0.0.1:8081',
        changeOrigin: true
      }
    }
  }
})
```
